## Product direction (agreed scope)

- **Single seller MVP**; teams/orgs deferred to a later phase (schema leaves hooks like nullable `organization_id`).
- **Multi-quantity inventory:** partial sales **decrement** stock on other linked listings; full delist/zero when canonical qty hits zero.
- **30-day relist:** scoped to **eBay only** in MVP; other platforms later if APIs/policy allow.
- **Depop:** **blocked** until partner API credentials; UI/adapter stub only.
- **Poshmark & Mercari:** hybrid / manual fetch and publish flows.
- **Subscription-ready:** Supabase Auth + Stripe-shaped tables (`customers`, `subscriptions`, `plan_tier`); AI gated on paid/active plan.
- **Nifty.ai** referenced as workflow inspiration only (not a clone).

## API ‚Äî notable behavior

- **Auth:** Bearer JWT from Supabase; `requireAuth` in route helpers.
- **OAuth:** eBay + Shopify URL endpoints (`/api/oauth/ebay/url`, `/api/oauth/shopify/url`) return JSON `{ url }` for SPA redirect; callbacks on `/oauth/ebay/callback`, `/oauth/shopify/callback`.
  - **eBay:** Scopes include `sell.inventory`, `sell.account`, and **`commerce.identity.readonly`**. After token exchange, the callback calls eBay **Commerce Identity** `GET ‚Ä¶/commerce/identity/v1/user/` (`apiz.ebay.com` / `apiz.sandbox.ebay.com`) and stores a JSON blob (username, userId, accountType, marketplace) in `integration_credentials.shop_domain` for UI display. Redirect after success: `/integrations?ebay=connected`.
  - **Disconnect:** `DELETE /api/integrations/:platform` removes stored credentials for that platform (scoped with **`user_id`** + **`platform`**; valid `platform` values match `PLATFORMS` in `@inv/shared`).
  - **eBay consent URL** includes **`prompt=login`** so eBay shows sign-in / account selection even when the browser still has an eBay session ‚Äî needed to switch accounts after disconnect. [eBay consent request](https://developer.ebay.com/api-docs/static/oauth-consent-request.html).
- **Integrations list:** `GET /api/integrations` returns `connections` (`platform`, `shop_domain`, `updated_at` per row; eBay `shop_domain` may be JSON) and, when **eBay is connected**, **`ebay`** `{ inventoryLocationReady, inventoryLocationDetail? }` from **`getEbayInventoryLocationStatus`** (`lib/ebayInventoryLocation.ts`) ‚Äî **`inventoryLocationReady: false`** means no enabled **`GET /sell/inventory/v1/location`** row (publish will fail until Seller Hub has a warehouse/business location). Detail text is surfaced on **Integrations** page. **`GET /api/integrations/ebay/inventory-locations`** (eBay connected) returns **`{ ebayApiBase, sandbox, total, locations }`** ‚Äî full objects from eBay‚Äôs Inventory API ([Managing inventory locations](https://developer.ebay.com/api-docs/sell/static/inventory/managing-inventory-locations.html)); uses **`fetchAllEbayInventoryLocations`** (offset/limit pagination). **`POST /api/integrations/ebay/inventory-locations`** (JSON body: **`name`**, **`country`** (ISO-2), and either **`postalCode`** or **`city` + `stateOrProvince`**) creates a **WAREHOUSE** location via **`createEbayWarehouseInventoryLocation`** ([createInventoryLocation](https://developer.ebay.com/api-docs/sell/inventory/resources/location/methods/createInventoryLocation), [creating locations](https://developer.ebay.com/api-docs/sell/static/inventory/managing-inventory-locations.html#creating)); **`merchantLocationKey`** optional (server generates `wh-` + hex). Sends **`Content-Language`** from **`EBAY_MARKETPLACE_ID`**. Success ‚Üí **`{ ok, merchantLocationKey }`** (eBay returns **204**).
- **eBay business policies (for draft editor dropdowns):** `GET /api/integrations/ebay/policies` ‚Äî requires eBay connection; returns `fulfillmentPolicies`, `paymentPolicies`, `returnPolicies` as `{ id, name }[]` from Sell Account API (`fulfillment_policy`, `payment_policy`, `return_policy` for `EBAY_MARKETPLACE_ID`). Used to populate shipping/return/payment policy selects on the draft page; publish still falls back to the **first** policy of each type if none selected.
- **eBay category + condition + item aspects metadata (Taxonomy + Metadata APIs):** Uses **application** OAuth token (`client_credentials` in `ebayAccessToken.ts`); does **not** require the seller to have connected eBay in-app, but **`EBAY_CLIENT_ID` / `EBAY_CLIENT_SECRET` must be set.** Follows the flow in [Finding categories for a listing](https://developer.ebay.com/api-docs/sell/static/metadata/sell-categories.html): `GET /api/integrations/ebay/category-suggestions?q=‚Ä¶` ‚Üí Commerce Taxonomy **`/commerce/taxonomy/v1/`** `get_default_category_tree_id` + `get_category_suggestions` (use **`v1`**, not `v1_beta` ‚Äî older path returns **404**). **`get_category_suggestions`** is documented as low-quality / not fully supported in **Sandbox**; use **production** (`EBAY_SANDBOX=false`) for realistic suggestions. **Leaf categories only:** Inventory publish fails with **25005** if `categoryId` is not a leaf; **`fetchCategoryLeafStatus`** uses **`get_category_subtree`** (`leafCategoryTreeNode === true`). **Suggestion expansion:** when Taxonomy returns a **parent** category, the API **walks the subtree** and replaces it with **leaf** suggestions (caps apply) so the combobox does not offer non-listable IDs. `GET /api/integrations/ebay/item-condition-policies?categoryId=‚Ä¶` ‚Üí Sell Metadata `get_item_condition_policies` with `filter=categoryIds:{id}`. **`GET /api/integrations/ebay/item-aspects?categoryId=‚Ä¶`** ‚Üí Taxonomy **`get_item_aspects_for_category`** (leaf `category_id`); returns required/optional aspects, `FREE_TEXT` vs `SELECTION_ONLY`, and allowed values for dropdowns. Metadata returns **`conditionId` as numeric strings** (e.g. `1000`, `3000`); Inventory API expects **ConditionEnum** strings (`NEW`, `USED_EXCELLENT`, ‚Ä¶). **`ebayCategoryApi.ts`** maps IDs to enums via **`normalizeEbayConditionForInventory`** from `@inv/shared`; **`draftValidation`** and **publish** (`drafts.ts`) normalize before validate/send. Helpers also in `lib/ebayLocale.ts`.
- **Webhooks:** Shopify HMAC (raw body preserved via JSON parser), eBay stub by user id path, Stripe subscription updates (when configured).
- **Sale sync:** `propagateSaleQuantity` + idempotency table; multi-qty helpers in `lib/qty.ts`.
- **Adapters:** eBay, Shopify (REST/HTTP), Depop blocked, Poshmark/Mercari hybrid.
- **Sync:** `POST /api/sync/:platform` for fetch/reconcile (eBay/Shopify); **`POST /api/sync/all`** runs sync for every connected integration (Inventory **Refresh** uses this).
- **AI:** `POST /api/ai/generate-drafts` ‚Äî Google Gemini (`gemini-2.5-flash`); **requires** active paid tier + `GEMINI_API_KEY` (free tier available at https://aistudio.google.com/). Vision uses up to **`DRAFT_EDITOR_INITIAL_PHOTO_COUNT`** (5) images from `inventory_images` order. Per-platform JSON still includes `title`, `body`, `price_hint_cents`, `tags`. **eBay** responses additionally ask the model for **`category_id_hint`** (US leaf category id digits when inferable) and frame **`price_hint_cents`** as a multi-channel resale estimate (not live sold comps). When **`EBAY_CLIENT_ID` / `EBAY_CLIENT_SECRET`** are set, the API enriches the eBay draft with the **first Taxonomy suggestion** from the listing title (overwriting `category_id_hint` and setting **`category_name_hint`**) when the call succeeds.
- **Jobs:** eBay relist scheduler (hourly) records `relist_due`-style events; full withdraw/republish can be extended later.
- **Phase 2 placeholder:** `GET /api/phase2/status` (roadmap JSON only).
- **Inventory list:** `GET /api/inventory` includes **`photo_count`** per item (nested `inventory_images(count)`).
- **Platform listings:** `GET /api/platform-listings` ‚Äî optional **`status`** and **`platform`**. **`status=live`** matches **`active`** (published from app) **or** **`pending_link`** (synced from marketplace before inventory link). **`status=active`** still filters to **`active`** only if you need it. Response includes **`listing_title`**, **`listing_image_url`** when set, and **`inventory_items ( id, title, sku )`** for list UI.
- **Listing drafts (extended):**
  - `GET /api/listing-drafts` ‚Äî list; optional `?platform=`, `?inventory_item_id=`, **`unpublished_only=true`** (only rows with **`published_listing_id` IS NULL**); embeds `inventory_items` + `inventory_images` for list responses (nested images include **`file_updated_at`** for thumbnails).
  - `GET /api/listing-drafts/:id` ‚Äî **detail** for the draft editor: anchor draft, eBay sibling row, all platform siblings for the same `inventory_item_id`, inventory row, photos (**`id`, `storage_path`, `sort_order`, `file_updated_at`**), integration rows.
  - `PATCH /api/listing-drafts/:id` ‚Äî merges into the **eBay** `listing_drafts` row‚Äôs `payload` (including nested `editor` for multi-platform form state). **`editor.ebay` (and other platform keys) are deep-merged** so partial saves keep nested `listing` / `shipping` / `pricing` objects. Register route order: `GET :id` before `GET` list (no param).
  - `POST /api/listing-drafts/:id/publish` ‚Äî body `{ platforms: string[] }`; runs **`validateDraftEditor`** from `@inv/shared` (for eBay, server loads **required aspect names** via **`fetchItemAspectsForCategory`** when OAuth is configured, and passes **`ebayRequiredAspectNames`** so missing item specifics fail validation before publish). Loads **`inventory_images`** with **`.order("sort_order")`**; selected image URLs append **`?v=<ms>`** from **`file_updated_at`** when present (cache-friendly after in-place file replace). **eBay:** `publishEbayListing.ts` (fulfillment/payment/return policies ‚Äî user-selected ids or account defaults, inventory location, **leaf category** check, **required item aspects** check vs Taxonomy, inventory item with optional **`product.aspects`** (from `editor.ebay.listing.itemAspects` ‚Üí string arrays per localized aspect name) and optional **`product.packageWeightAndSize`**, offer with optional **`listingPolicies.bestOfferTerms`**, publish) + inserts `platform_listings` and sets **`listing_drafts.published_listing_id`** on the matching **`ebay`** draft row. **Shopify:** same pattern ‚Äî insert **`platform_listings`** + set **`published_listing_id`** on **`shopify`** draft row. **`condition`** sent to eBay is **`normalizeEbayConditionForInventory(editor.ebay.condition)`** (Inventory enums only). Description can be prefixed with **Color:** from `editor.ebay.listing.color`; optional **SKU** overrides generated `inv-‚Ä¶` sku when valid. **eBay REST:** all Sell calls send **`Accept`**, **`Accept-Language`**, and JSON writes send **`Content-Language`** (BCP-47 from `EBAY_MARKETPLACE_ID` via `ebayLocale.ts`) to satisfy Inventory/Account APIs. **Offers:** if an offer already exists for the SKU (e.g. after a failed publish), the flow **updates** via `PUT ‚Ä¶/offer/{offerId}` instead of `POST ‚Ä¶/offer` (with fallback if create returns error **25002** duplicate-offer race). **Shopify:** `publishShopifyListing.ts` (Admin REST product create); Depop/Poshmark/Mercari return not available in-app.
- **eBay token refresh:** `lib/ebayAccessToken.ts` (refresh + `getValidEbayAccessToken` for publish; **`getEbayApplicationAccessToken`** for Taxonomy/Metadata). Refreshed user tokens are not yet persisted back to `integration_credentials` after publish (follow-up if long sessions).
